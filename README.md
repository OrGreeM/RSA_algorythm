# RSA_algorythm

Термінальний чат з шифруванням повідомлень. RSA використовується для обміну симетричним ключем, сам чат шифрується XOR-ом, цілісність перевіряється через SHA-256.

Як запустити?🤔🤔🤔🤔


1. Відкрийте три термінали в папці з проєктом.

1.1 В першому терміналі напишіть:
python server.py

1.2 В другому:

python client.py alice # <-- Ваше ім'я, а може і не ваше ім'я

1.3 І в третьому:
python3 client.py bob # <-- Ше якесь ім'я

P.S. Перший запуск кожного клієнта займає кілька секунд — генерується RSA ключ. Це нормально.

2. Тепер можна переписуватись...


3. Вихід — Ctrl+C.

Як це працює? 🤔🤔🤔🤔🤔

Коли клієнт підключається:

Генерує собі RSA пару ключів (1024 біт).
Відправляє серверу своє ім'я.
Відправляє серверу свій публічний ключ.
Сервер шифрує публічним ключем клієнта свій симетричний ключ і надсилає назад.
Клієнт розшифровує цей симетричний ключ своїм приватним RSA-ключем.

Далі все спілкування йде через симетричний ключ — RSA більше не використовується.
Коли клієнт відправляє повідомлення:

Рахується SHA-256 хеш від тексту.
Текст шифрується симетричним ключем (XOR).
Склеюється хеш + шифротекст, кодується в base64, летить через сокет.

Коли клієнт отримує повідомлення:

Декодує base64.
Відрізає перші 32 байти — це хеш.
Розшифровує решту симетричним ключем.
Рахує хеш від розшифрованого тексту і порівнює з отриманим.
Якщо хеші збігаються — виводить повідомлення. Якщо ні — пише що цілісність порушена.
